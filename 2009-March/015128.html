Comment(by BeOSR):

 To be semantically correct, the status line would show the number of
 *files*, not *items*, and their cumulated disk space. As folders and
 symlinks aren't really files but placeholders, this should be clear.

 Humdinger's point of showing the number of selected files and their total
 size is absolutely correct, so there'd be 2 views:[[BR]]

 Basic view, nothing selected:[[BR]]
 15 files | 5.4 Mb[[BR]]

 With some files selected:[[BR]]
 6 files selected | 1.2 Mb

 Finally, it would be handy to have some indicator in the status line that
 parent folders can be browsed from that point, a lot of people don't know
 about this. Maybe use two dots to indicate parent directory or a dot and
 an up arrow or something? The very long horizontal scrollbar is a waste of
 space in most cases anyway.[[BR]]
 ^. | 15 files | 5.4 Mb < [scrollbar] >[[BR]]
 .. | 15 files selected | 5.4 Mb < [scrollbar] >
