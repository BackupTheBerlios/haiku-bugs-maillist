Comment(by luroh):

 Hang on, I think I know what is going on here; Haiku picks up the presence
 of an old deleted Haiku partition on my SATA drive (an old revision number
 in About gave it away!):[[BR]][[BR]]
 1. Booting starts from the USB stick.[[BR]]
 2. At some point during boot, the system detects an old deleted Haiku
 partition on the SATA drive and decides to set it as /boot. This explains
 why the system loaded slowly during bootsplash but fast once it reached
 Desktop.[[BR]]

 To avoid 2 from happening, I not only had to delete the old Haiku
 partition on the SATA drive, but reformat the partition with an alien file
 system to completely rid it from its BFS-edness.[[BR]][[BR]]
 Perhaps we should close this ticket as invalid and start fresh, as it
 seems to have morphed into something completely different than it
 initially was.[[BR]][[BR]]
