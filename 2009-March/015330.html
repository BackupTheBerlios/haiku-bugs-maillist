Comment(by umccullough):

 I still think this would be better served by adding a check somewhere in a
 Jamfile that is always evaluated when the user invokes jam on the Haiku
 sources :)

 An update to jam does not require reconfigure, nor does configure require
 jam to proceed. Thus, it should *not* be a requirement to have jam
 installed to build the cross tools (as it currently isn't).

 It's highly likely that any future fixes to jam and this version check
 will go unnoticed by many because they will just svn up && jam -q without
 ever encountering this check.
