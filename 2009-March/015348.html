 The Twitcher would be easier to immediately understand if it had a
 vertical column of windows. This would be a more effective visual
 representation of the current set of windows than the current single-item
 "list" of windows, the one with the up/down triangles.

 I´m not sure on the exact behavior, or graphics, like whether all apps
 should have their windows shown or only the selected application.

 Either you have a visual cue to signify which window is selected,
 (probably a larger-size icon like the app icon) or you shift the windows
 around some way.

 I think I would prefer if the apps and windows are sorted alphabetically
 and don´t jump around at all, in neither orientation. Instead the
 selection-cue moves. (Unless there are too many apps or windows to fit.)

 In spite of the sorted presentation, tapping ctrl-tab would still let you
 ping-pong between any two windows.

 There could be several Twitcher modes, e.g. if some people wanted more
 Windows-like ping-ponging between, say 3 or 4 windows.
