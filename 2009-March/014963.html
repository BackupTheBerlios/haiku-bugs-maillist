Comment(by VinDuv):

 Are you using the Haiku double-track (ISO/BFS) boot CD (created with jam
 haiku-boot-cd / jam haiku-image) or the new single track ISO CD (created
 with jam haiku-cd) ? I encountered the same issue with the latter.

 It seems that the app_server (and other servers launched before it, except
 registrar) fails to launch. The "teams" KDL command only shows 4 teams :
 sh, kernel_team, registrar and waitfor ("/bin/waitfor picasso").

 Disabling the write_overlay for the boot volume (see
 source:haiku/trunk/src/system/kernel/fs/vfs_boot.cpp at 29261#L447) fixes the
 issue.
