Comment:

 Removing form the R1/alpha1 milestone, for the same reasons as #1822.
 Actually, write support is there for the features we need (initializing
 partitions). The most important problems are:

 1) Retrieving possible disk system candidates for completely empty
 volumes. Currently, an empty harddrive can be initialized with BFS, but
 the BDiskDevice methods to retrieve disk systems do not work, since a
 proper object hierarchy is not being setup in the kernel.

 2) Write support in the Intel Extended Partition. The Intel Partition Map
 write support could actually work already, but needs proper testing.
