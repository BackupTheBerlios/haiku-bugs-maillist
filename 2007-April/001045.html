 Trac 0.11 has changed quite the roadmap code, so we need to create a
 completely new patch that groups components into the top-most component.
 For example, all tickets for Website/Trac and Website/CMS should be
 counted to Website and only one entry should be shown. You can see this
 behavior with our current Trac release (0.10) by looking at milestone:R1.
 As you can see, on the right the progress bars only appear for the parent
 components, but not the subcomponents. When you click on the progress bars
 you get search results matching the subcomponents, too (via "begins with",
 which was added manually).
