 We currently use Gallery2 which was a performance bottleneck when we got
 hit by Slashdot. We've done quite a few optimizations since then, but
 still, G2 create more DB load (more connections, more queries) and it
 transfers files via PHP instead of statically which is about 10 times
 faster.

 Either write a new (or extend an existing) gallery module which does what
 we need and maybe also has slideshows or use Drupal's default gallery
 (which doesn't work with the category module, see #1136). In any case, our
 current image upload module must be supported (I don't know if that needs
 extra work).

 Note, #1132 will be affected by this.
