 This may be a bit controversial, but I'll give it a go anyway...

 Most documentation for poll() states that programs using poll() should
 #include <poll.h>.  However, this rule seems to be more honoured in the
 breech than the observance, with a lot of 3rd party code #including
 <sys/poll.h>.  Haiku does not have <sys/poll.h>, so 3rd party code that
 #includes <sys/poll.h> needs to be modified to compile.

 Posix OSs that I have seen (Linux, Solaris and NetBSD) have the main
 poll() header in <sys/poll.h>.  <poll.h> either #includes, or is a
 symbolic link to, <sys/poll.h>.

 I have attached a patch that makes Haiku emulate Linux/Solaris semantics:
 <poll.h> #includes <sys/poll.h>.  This patch should not break existing
 Haiku/BeOS software, and will allow 3rd party apps to be ported with less
 modifications.

 Any chance of it being applied?
