Comment:

 You are right that MediaPlayer does not save the volume setting.

 Since there can be multiple MediaPlayer windows I think we need a per file
 volume setting, which I figure could be stored as an attribute of the
 files in question. So I did some research and indeed the R5 media player
 stores the volume setting (and quite a few other settings) as attributes
 of the media file:

     Int-32          4           be:mediaPlayer_miniMode
 0x52454354         16              be:mediaPlayer_frame
      Float          4             be:mediaPlayer_volume
     Int-64          8            be:mediaPlayer_inpoint
     Int-64          8           be:mediaPlayer_outpoint
       Bool          1         be:mediaPlayer_dropFrames

 So I think we need to do something similar.
