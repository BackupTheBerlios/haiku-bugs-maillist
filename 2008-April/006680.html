Comment (by bonefish):

 Actually the origin of B_COMMAND_KEY is the Command (Apple) key (BeOS ran
 first on Mac clones). And the equivalent location on PC keyboards is the
 Alt, although Windows (and Linux by default) use the Control key for
 common shortcuts. Which is the only reason for the Alt/Ctrl option in the
 Menu preflet. If the Alt key doesn't map correctly to the Command key on
 Apple keyboards, that should be fixed. Not sure where exactly, though
 (keymap, driver?).
