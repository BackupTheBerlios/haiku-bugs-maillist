 Ever since updating my apr 0.9.x builds for i586-pc-haiku (and thus
 getting libapr.so) I get this whenever something's trying to use
 libapr.so:

 {{{
 elf_resolve_symbol: could not resolve symbol 'pthread_sigmask'
 resolve symbol "pthread_sigmask" returned: -2147478780
 runtime_loader: troubles relocating: 0x80001304 (image: libapr.so.0)
 }}}

 I cannot remember seeing such an issue while using my own patched libtool
 with {{{--build=i586-pc-beos}}}, some time before the Optional Packages. I
 checked for occurrences of beos in configure.in etc. so I don't think it's
 something I've forgotten.

 {{{pthread_sigmask}}} is being declared in the {{{signal.h}}} header but
 apparently only referenced in the Posix test suite; I cannot find an
 implementation anywhere. Is it missing or some magic going wrong here?
