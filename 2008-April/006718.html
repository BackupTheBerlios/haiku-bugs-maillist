Comment (by axeld):

 Yes, that's a known problem; it happens because of the way the current
 value is computed - since the data source retrieval happens in the window
 thread, it can easily lose a value (for example when doing a screenshot).
 Even if it would not run in the window thread anymore (which would make
 this problem a lot less likely), it might still lose a value under heavy
 load.
