Comment (by mmlr):

 The mouse cannot work, as the host controller that you have is not
 supported by any driver right now. If it works then this is only because
 of legacy emulation. That you get messages from EHCI indicates that EHCI
 is working properly, but as the device you're attaching is low or
 fullspeed, EHCI will just hand over control to the companion host
 controller (OHCI). This will then not do anything with the device as there
 is no driver for OHCI.
 What concerns me more is that the serial logs you are posting are
 completely distorted. Are those actual serial logs or are those syslogs?
 In any case it looks to me like some synchronization / locking is not
 working, so messages from different places get mixed up completely. Maybe
 someone else can have a look and maybe has an idea as to what could be
 going wrong here?
