 using r24979 on real hardware.

 start a large download ( the new firefox from bebits ) or start to expand
 a large zxip ( the same file downloaded using BeOS max ) the operation
 continues for 10 to 15 seconds then just stops. Using the process
 controller, one can see that the kernel team page writer thread is using
 all the cpu cycles. No file operations can be done at this time. the
 system is not frozen, but the only thing that can be done is to reboot.
 any running apps are still responsive, but no new apps can be launched and
 nothing can be saved.
