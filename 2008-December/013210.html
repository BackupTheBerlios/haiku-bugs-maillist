 I installed DarkWyrm's Paladin, a BeIDE replacement. I'm running into some
 filetype issues that don't exist in BeOS.
 When I create a project file, its file type is messed up.

 To reproduce:
 - Install Paladin 1.0 RC from Bebits
 - Run it and create a new project
 - Save it and exit
 In Tracker, the file home/MyProject/MyProject.pld is identified as a
 generic file. See BeforeOpening.png. However, when I catattr the file, it
 returns a Paladin file type:
 {{{
 ~/projects/MyProject> catattr BEOS:TYPE MyProject.pld
 MyProject.pld : string : text/x-vnd.dw-Paladin.Project
 }}}
 Paladin installed the FileType association correctly as far as I can see.
 (see PaladinFileType.png)
 Double clicking the file opens it in StyleEdit. It also doesn't have the
 correct icon.

 When I Identify the file, it is identified as a Text File (text/plain).

 With the File Type addon I changed its type to text/x-vnd.dw-
 Paladin.Project. Now this type is also seen by catattr, and also by
 Tracker ("Paladin Project" as the kind).

 However, when I open the project and close it, it is saved and the same
 issue as above occurs again.

 I don't know where tracker gets its information, but if catattr BEOS:TYPE
 returns text/x-vnd.dw-Paladin.Project, it should find the type to be a
 Paladin Project.
