 Just to be clear: The BeOS API which Haiku inherits (and will keep and
 extend for the foreseeable future) requires error codes to be negative, so
 that e.g. a method's negative ssize_t return value does not only indicate
 that an error occurred, but does directly encode the error code. This
 concept is consequently used in the BeOS API and we can't change it
 without breaking pretty much all existing code.
