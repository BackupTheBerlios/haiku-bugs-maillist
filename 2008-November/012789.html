Comment(by Adek336):

 Replying to [comment:51 Adek336]:
 > _fbsd_uninit_driver will delete the devices which it finds in gDevices.
 I'm not sure if this is a problem, but if_free now removes the entry from
 gDevices so probably the device will never get deleted after calling
 if_free; on the other hand, if the driver doesn't call if_initname, no
 entry in gDevices is created so the device probably can't be deleted
 neither.

 Using ld to wrap around malloc,calloc,realloc and free in the driver
 binary I can see that there are no memory leaks with the
 3com+reproduce.diff, that is, after going through probe / try attach /
 detach-after-failure-in-attach.

 That is a rough indication, that my patch doesn't infere with deleting of
 the devices.
