 While running a configure script in tight memory situation, I got an
 application crash. Clicking on the "Debug" button of the error dialog,
 I was presented a gdb session the shell that was executing the
 configure script. At the top, function fork().

 To reproduce:

 0. Start Haiku in QEMU with 256 MB RAM allocated to it,

 1. Open a Terminal.

 2. $ wget http://www.haible.de/bruno/gnu/testdir3.tar.gz

 3. $ tar xvfz testdir3.tar.gz

 4. $ cd testdir3

 5. $ ./configure --host=i586-pc-haiku --build=i586-pc-haiku
 --prefix=/boot/home/config CPPFLAGS="-Wall" 2>&1 | tee log1

 6. Wait a couple of hours.
