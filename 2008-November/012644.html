Comment(by tigerdog):

 New Info:
 I am now able to use nvidia accelerated graphics here, so I have tested
 different modelines in ProposeDisplayMode.c.  The problems I see are
 definitely caused by the 1920x1200 modeline  introduced by revision 22955.
 I am able to get a satisfactory display using the older modeline.
 However, I also tested using a modeline modeled on windows and Linux.

 {{{
 /* 16:10 panel mode; 2.304M pixels */
 { { 193160, 1920, 2048, 2256, 2592, 1200, 1201, 1204, 1242,
 T_POSITIVE_SYNC}, B_CMAP8, 1920, 1200, 0, 0, MODE_FLAGS}, /*
 Vesa_Monitor_ at 60Hz_(1920X1200) */
 }}}
 This produces a more stable display, at least with my monitor and card.
