Comment(by axeld):

 I've tracked down and fixed a pretty bad interaction between the file and
 the block cache in r28517 - freed blocks could be written back after they
 were claimed by the file cache.

 This should also fix this bug, please retry.

 BTW: the problem with "blocks already set" is that you don't know the
 second file when that happens (as two files are likely corrupted due to
 this). You can use "bfswhich" of the BFS-tools to find out about that,
 though.

 In any case, it shouldn't affect new files.
