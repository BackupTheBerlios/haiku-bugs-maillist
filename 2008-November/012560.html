 ISO C 99 section 7.19.9.2 specifies that "a successful call to the
 fseek function undoes any effects of the ungetc function on the
 stream". Same in POSIX, see
   http://www.opengroup.org/susv3/functions/fseek.html

 Attached is a test case showing that this is not fulfilled in Haiku.
