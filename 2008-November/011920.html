 Mounting a CD or DVD then pressing the eject button to remove it properly
 removes the entry from Tracker, but according to the 'ls' command in
 Terminal, the entry remains.

 When re-inserting the disc, a new folder in the root filesystem node is
 created with the same name and an incrementing number. This is entirely
 reproducible (on real hardware at the very least).

 Unmounting the volume with Tracker avoids this problem and ejects the CD
 happily.

 Perhaps there's an issue with a lock on the folder when the system detects
 the CD is removed? I'm unable to test this with USB media.
