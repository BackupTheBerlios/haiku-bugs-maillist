 With the recently updated bison package, ftpcmd.y -> ftpcmd.c no longer
 works correctly. It generates a symbol table, but then puts the enum
 definitions for the tokens used in said table later in the file, as a
 result of which gcc fails to build the file. This still works correctly on
 Linux and FreeBSD though, both of which are using bison 2.3 here.
