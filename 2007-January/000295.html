Comment:

 It turned out the driver was adding ETHER_ALIGN (2 byte) to the 2048 byte
 rx/tx buffers. Thus asking for 2048 byte memory map at offset 2 could
 result in two page_entry descriptions. This wasn't a problem in BeOS R5,
 as it didn't return an error, and only the first description is needed.
 But it's still a driver bug, as the haiku behaviour is better. I modified
 the driver to only ask for the memory map of 2046 bytes.

 Fixed in r19992.
