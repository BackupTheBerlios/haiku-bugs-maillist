Comment(by sil2100):

 I am sorry this took so long. I made this patch a few days ago, but the
 tracker was down, and I had to leave for a trip later that day. But I
 finally found a place to connect to.

 This patch does the thing in mention a bit differently. Since the maximum
 number of buttons in a BAlarm is 3, I couldn't add a 'Yes to all' button -
 it's not really a good idea anyway, since the number of buttons would make
 the alarm less readable. Instead, when a conflict appears, a second dialog
 box appears asking the user if the PackageInstaller should remember the
 previous choice for all conflicts in the future. Right now the installer
 will ask only once, but a 'Ask later' button is possible - although I
 can't find as much use to such thing, it would probably just unnecessarily
 bloat the alert.

 I also exported the user asking mechanism to the View instead of the inner
 structures. This way seems much better. Tested on r31688.
