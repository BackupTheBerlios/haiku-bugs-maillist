Comment(by leavengood):

 I definitely agree with this, but there is a problem because AFAIK, the
 B_BACKGROUND_APP flag (which keeps the app from showing in the Deskbar) is
 set as a resource and cannot be changed at runtime. Since sometimes we
 want to run the Screenshot app like a normal application (to set options
 in the GUI), it doesn't make sense to have B_BACKGROUND_APP set
 permanently.

 Of course I could be wrong, and even if I am right, this could be a new
 feature in Haiku to change the app_info at runtime (at least this
 property.)
