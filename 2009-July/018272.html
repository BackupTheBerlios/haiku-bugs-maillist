 If there is a Tracker window in another workspace and it is selected in
 the list, many times Switcher will not change to that workspace.

 The simplest way to reproduce is to have a single Tracker window in
 another workspace and try to switch to it. Having a couple Tracker windows
 seems to fix the problem, sometimes. It is quite inconsistent so I assume
 it is a bug.

 I don't know if somehow the Desktop window is screwing things up for
 Tracker, because other apps do not seem to have this problem. Also when
 the other window is selected and the workspace is not changed, the Desktop
 has the focus, hence my suspicion.
