Comment(by kallisti5):

 I vote to leave it in.

 What happens is on the first missing relocation we return an error
 instantly throwing us back to the bootloader and causing the end user to
 not see the error message.

 With the _BOOT_MODE def in there we will try and push on if it is the
 kernel booting showing all the relocation errors and then freeze up when
 trying to jump into the kernel.

 Missing relocation errors still may occur in the future and this is a
 handy way to fish them out.
