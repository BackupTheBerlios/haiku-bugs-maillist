 The disconnect handling for file descriptors has several issues:
  - Disconnected file_descriptors are leaked (put_fd() sets the ref_count
 to -1).
  - There are race conditions, particularly in put_fd().
  - Disconnected file descriptors can be dup()ed and copied on fork(),
 messing with the mangled ref_count.
