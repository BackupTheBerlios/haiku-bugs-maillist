Comment(by leavengood):

 I've been researching tabs lately for the browser project and I looked
 over the BTabView code, and it indeed will stop detaching views if a
 BCardLayout is present. It just delegates to the card layout to hide and
 show the views as the tabs are selected.

 So as Stephan says switching to using layout management (well at least
 using a card layout) would probably fix this issue. Of course the terminal
 GUI does some off-by-one manual layout tricks to make the scrollbar look
 better and I am not sure how that would be affected by using layout
 management. Guess we will see if and when someone tries.
