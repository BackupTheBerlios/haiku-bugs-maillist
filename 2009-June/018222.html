Comment(by jackburton):

 I guess it's correct like it is. Those mouse messages are sent by the
 app_server, so they are remote messages. The fact that using
 B_LOCAL_SOURCE you get some messages is because, IIRC, BWindow sends some
 of these messages itself.
 The bebook is also very clear about that: to receive messages from
 app_server, you have to use B_REMOTE_SOURCE (or B_ANY_SOURCE), so I'd be
 inclined to close this bug as invalid.
 Did that code work on beos, by chance ?
