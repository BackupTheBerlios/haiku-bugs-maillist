 Thanks Stippi!!

 I came to my fix by reading the docs on fs_write_attr(). It says that it
 is an incomplete implementation and that pos is supposed to be ignored. It
 also says that you can basically write to an attribute only once as the
 next write will completely replace the attribute contents, even if the old
 data is larger than the new. In the case of the loop, it should actually
 never go through the loop more than once.

 If you change this behavior in fw_write_attr(), you could break a lot of
 existing apps.
