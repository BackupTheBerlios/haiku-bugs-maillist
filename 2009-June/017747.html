Comment(by idefix):

 I investigated this further and it doesn't matter if the view is added
 with the mouse or the keyboard.

 What you do need is Haiku running in VMware on a slow computer (PII @
 400MHz) and the time interval set to a low value (less than 100ms),
 causing the processor to be fully utilised by ActivityMonitor. It will
 then reliably crash every time you add a view.
