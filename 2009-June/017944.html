 The cp command does copy attributes by default, but it doesn't copy empty
 ones. This is due how the copy_attributes routine is written in the copy
 backend of cp (src/bin/coreutils/src/copy.c). Comparing the source of
 copyattr's copy_attributes it is written in a way to ensure that empty
 attributes are created correctly.
