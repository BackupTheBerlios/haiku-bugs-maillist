 Priority:  critical           |   Milestone:  R1

 When a file is unlinked/deleted, you must only make it unavailable to  the
 user (ie. it must not appear in file listings anymore, and cannot be
 opened).

 However, its space on disk must not be deleted until the kernel no longer
 needs the file (it will then call fs_remove_vnode()). Until that one is
 called, an application with an open file handle must still be able to read
 from that file, and write to that file as if it would still exist.

 Please check if the file system remains valid when actually doing so - in
 that case, the priority could be put back to "high". As is, I wouldn't
 enable read/write for the file system yet.
