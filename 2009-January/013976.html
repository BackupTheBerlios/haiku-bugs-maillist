 So select a target partition, choose more options, open DriveSetup from
 the button, close it, now the target partition is not the one you chose
 but the first from the list. This may be a tragic mistake for someone not
 cautious!

 I understand that after potentially changing the partition layout with
 DriveSetup the target partition selected list has to repopulate so you
 can't easily preserve the position on that list. So I guess the first
 position on the list should be a fake position, something like "Please
 select a destination".

 That would also help with the problem, that after starting up, the
 Installer has the first availlable partition selected: if you accidentally
 pressed enter, you'd have it rewritten!
