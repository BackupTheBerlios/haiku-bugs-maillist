Comment(by euan):

 Still having problems with my HP 6715b laptop.  Found the following info
 regarding linux that might shed some light on the evil details of this
 laptop:

 http://bugzilla.kernel.org/show_bug.cgi?id=11715#c11
 http://www.nabble.com/-PATCH--x86:-remove-superfluous-
 dmi_ignore_irq0_timer_override-quirks-td20011385.html

 Is the above bios issue something of interest to Haiku's setup?  This info
 was in the linux DMI quirks file (2nd link):

 HP laptops which use a DSDT reporting as HP/SB400/10000, which includes
 some code which overrides all temperature trip points to 16C if the INTIN2
 input of the I/O APIC is enabled.  This input is incorrectly designated
 the ISA IRQ 0 via an interrupt source override even though it is wired to
 the output of the master 8259A and INTIN0 is not connected at all.  Force
 ignoring BIOS IRQ0 pin2 override in that cases.

 Applies to the SB600 chipset too apparently.
