Comment(by axeld):

 In your case, you should boot into Linux, make sure you have qemu
 installed, and then run it in a terminal like this:
 {{{
 $ sudo chmod o+r /dev/hda
 $ sudo chmod o+r /dev/hda16
 $ qemu -m 512 -serial stdio /dev/hda
 }}}
 You should then select to boot Haiku from your normal boot menu in the
 qemu window, and try to enter the boot menu as usually.

 Configured like this, qemu will dump the serial debug info into the
 terminal - and you can post this to this bug (if the bug happens there,
 too) :-)
