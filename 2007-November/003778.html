Comment (by chrisjp):

 While I don't get the same same stack trace, I get the same error on my
 macbook and desktop with GMA950 integrated (intel_extreme) if the hardware
 cursor is enabled.  Seems to be a problem with the hardware allocation of
 the cursor memory.  By setting the boolean in the driver (bHardwareCursor
 ? something like that) to false before the allocation, it fixes the
 problem.

 Hope that helps somewhat.
