 In VMWare I don't know what the problem is.

 The non-emulator booting freezes at the haiku logo as the new SATA driver
 doesn't have the vendor/product ID used in the macbook.  Updating the
 ahci.c file:

         { 0x8086, 0x27c3, "Intel ICH7R (RAID mode)" },
 +       { 0x8086, 0x27c4, "Intel ICH7 (AHCI Macbook)" },
         { 0x8086, 0x27c5, "Intel ICH7-M (AHCI mode)" },

 will allow haiku to boot, however due to other usb related uhci bugs,
 haiku will not function properly.

 Currently we can use external usb mice/keyboards unreliably, but I'm
 working on figuring out the usb  problem and hopefully will put together a
 touchpad driver for the macbook :)
