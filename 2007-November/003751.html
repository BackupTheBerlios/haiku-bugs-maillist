Comment (by bonefish):

 tls_get()/tls_set() must not be called with an invalid key (they are
 implemented inline in BeOS, so we can't really do much about this). If the
 pthreads implementation relies on tls_get() behaving a certain way, it is
 broken.

 That aside, we don't clear the TLS slots upon thread construction, which
 we probably should do. The related code is in
 src/system/kernel/arch/x86/arch_thread.c:arch_thread_init_tls(), if anyone
 wants to do that.
