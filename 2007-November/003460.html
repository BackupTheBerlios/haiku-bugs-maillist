Comment (by darkwyrm):

 Sounds about right -- theming, but not skinning. It would be about the
 level of customizability as Windows 98 officially had plus the choice of
 decorators. Ideally, it would combine a number of the other applets,
 hopefully the Fonts and Menu applets and possibly Backgrounds. It should
 provide sufficient power to customize for most people. Actual skinning
 (complete customization of the look of controls) not only sucks up
 performance, but would also require significant changes to the Interface
 Kit and possibly the app_server. Everything I'm talking about is already
 pretty much possible in the system. Full-on theme support (a la BeTheme)
 is being left for a third part opportunity, though anything is possible
 for R2. Appearance hasn't been documented officially, but has been
 discussed somewhat on some of the mailing lists, primarily the Interface
 Kit list IIRC.
