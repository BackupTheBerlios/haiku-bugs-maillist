 Using r28290, I can generate this "Segmentation violation" error in a
 Terminal by running vlc repeatedly (usually fails within 2 or 3 tries).

 Example (success first time, failure second time):

 {{{
 /boot/apps/vlc-0.8.6c> vlc
 VLC media player 0.8.6c Janus
 scrollview BView::InitData(): resizing mode or flags swapped
 /boot/apps/vlc-0.8.6c> vlc
 VLC media player 0.8.6c Janus
 Segmentation violation
 /boot/apps/vlc-0.8.6c>
 }}}

 Only seems to be repeatable when both cores are enabled, when only a
 single core enabled (by disabling one with pulse), I cannot make it occur.

 Quick machine specs: Core 2 Duo E6850 (2.66ghz), 4gb RAM, 500gb SATA disk
 (using ata bus_manager), intel G33 video, rtl8168 network

 I ran with strace to generate the attached output with failure.
