 I have an Intel Core 2 Duo system, the NIC is a rtl8101 (id 8136).
 As soon as there is any network activity, the cpu activity goes to 100%
 (only for one cpu, the other is idle) and the system becomes completely
 unresponsive (yeah, not even F12 works anymore). Top shows that the thread
 "fast taskq" is getting all the cpu load.
 The driver uses the freebsd compat layer.
 Disabling one core makes the system work correctly.
