Comment(by jonas.kirilla):

 Replying to [comment:5 axeld]:
 > Eventually, the FreeBSD compatibility layer would need to allocate some
 memory upfront for the mbufs to make sure they don't easily run out.

 I don't know if it's possible with the compatibility layer (or perhaps
 already done), but some kind of interrupt mitigation would help a loaded
 system from receiving traffic it can't handle, limiting/dropping traffic
 before it enters the system and resources are wasted on it. It doesn't do
 any good for failed allocations, but it should help overall.

 OpenBSD are looking at live shrinking a NIC's ring buffer based on how
 well mbufs reach userspace. (I hope I got that right.) I think Linux
 switches from interrupts to polling a NIC, beyond a certain threshold.
