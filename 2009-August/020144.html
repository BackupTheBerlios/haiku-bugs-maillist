Comment(by mmlr):

 Just a note: I've seen the same error message on OS X the other day. There
 a GPT is used and the MBR is not able to fully represent the partitioning.
 After syncing the GPT map with the MBR (using rEFIt) it worked though. The
 real fix would of course be to support GPT from our build system (or use
 native methods to get the offsets for that matter). I just wanted to
 mention it here, in case one of the problematic cases is actually using
 GPT. A fully protective MBR in GPT will for example always only contain a
 single partition spanning the whole disk, so this would never work.
