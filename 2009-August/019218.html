 I was wondering if it would make sense to move the fuse-fs initialization
 from FUSEFileSystem to FUSEVolume, and instead of having one server per
 fuse-fs linked to the fuse library, a fuse-userlandfs which would
 instantiate the fuse programs for each volumes. Of course then the mount
 command would look something like "mount -t userlandfs -p 'fuse sshfs
 user at host:' mntpt".
