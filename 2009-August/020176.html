Comment(by mmlr):

 Just realizing that BTW: The max packet size that is reported for the
 endpoint is only the max packet size the device can transfer per one USB
 transaction. This has no influence or meaning outside of USB. Meaning that
 if there is a max packet size of 8 bytes this does by no means mean that
 you can or should only schedule 8 bytes at a time. The only thing it means
 is that if you schedule more bytes that those it will (internally) split
 these up into multiple USB packets. With that context, is it possible that
 the event you are receiving simply doesn't fit into that buffer?
