Comment(by stippi):

 But showing "Unkown" for every partition is no worse than showing "Linux
 native" for every partition. In these situations, I believe users will
 rely more on partition size and layout, and that should be the same. I am
 not sure, but from what I remember, GParted sets the table entry to the
 equivalent of "Linux native" or "Linux swap" when creating the partition.
 It may still be that type even if you later initialize such a partition to
 FAT32 for example. In any case, the actual content type and the table type
 may be out of sync. That's why I don't think this feature would add a
 whole lot of value. It's quite easy to change, though, if you want to have
 a look. The table type is available via BPartition::Type(), while
 DriveSetup currently uses BPartition::ContentType(). See
 PartitionListView.cpp for more details. You could just add another column
 and provide a patch. But like I said, I think it will be more confusing
 than helpful (than you would have to make it obvious in the guy, which
 type is which and which is the true content type and which only the
 partition type marker from the MBR...).
