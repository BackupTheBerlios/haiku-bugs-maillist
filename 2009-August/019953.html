 SoundPlay has a plugin called QING that, when in full screen mode, uses
 BDirectWindow. When you enter this mode, it changes the resolution.

 If you leave full screen mode and then close SoundPlay, it is noticeable
 (at least with VESA) that the app_server tried to switch the resolution
 back to the original one. The problem is that the resolution is already
 the original one (the BWindowScreen was quit) and then the screen flickers
 unnecessarily.

 The app-server should first check if the resolution is not already the
 original one before trying to revert it.
