 As far as I can see in the userlandfs code, Mount() creates a volume which
 is mounted, and deleted if mount files, but Unmount() doesn't delete the
 volume, neither can I find any other places where it might be deleted.

 Another bug caused by this is that since fuse only allows one volume to be
 mounted, even after unmounting the current volume it is not possible to
 mount any other fuse-volumes without killing the userlandfs server hosting
 the related fuse filesystem.

 I'd suggest them to be deleted when they're successfully unmounted.
