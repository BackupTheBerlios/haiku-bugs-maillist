 BeOS R5 has a flag that Views can set - B_NO_POINTER_HISTORY - useful for
 when MouseMoved hooks take a lot of processing and can't keep up with the
 speed of new mouse messages.

 One example of where this is used is in the column view of Tracker, which
 exhibits slow resizing in Haiku (#160).

 I've written a small test app to compare R5 and Haiku behaviour, attached
 below. It shows B_NO_POINTER_HISTORY does not seem to have any effect in
 Haiku.
