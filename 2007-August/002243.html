Comment (by axeld):

 I had a look at it, but actually, it would be somewhat nice to solve this
 in the app_server. There is already the somewhat weird behaviour that it
 draws the view bitmap using the view's drawing and blending mode (it
 probably doesn't do this on BeOS), and it could be extended to blend them
 correctly then.

 Also, I'm not sure if it exists already, but is there a drawing mode that
 puts the current low color to a bitmap's transparent parts? If not, I
 think that one would be very practical to have (and not just for that
 case).

 What would you think about that stippi?

 Putting the code into Tracker would have the downside that you would need
 to rerender the bitmap whenever the desktop color changes (ie. because you
 use the same image on a different workspace with a different color).
