Comment:

 This seems to be a deadlock in the TCP implementation; there are kernel
 semaphores which you cannot interrupt. Can you provide a stack trace of
 the FTP thread? You can get this via "sc <thread-id>" in the kernel
 debugger. You could see the thread ID of the FTP thread using the "teams"
 command (in hex, so you'll have to add "0x" in front of the number before
 passing it to "sc").

 Both commands have to be entered in the kernel debugger which can be
 entered via the F12 key.
