 About the long execution time: Each test is performed eight times: four
 different color_spaces for flatten/unflatten and archive/unarchive. The
 tests are also not optimized. For example the bitmap for offscreen
 rendering is created for each test. At least under BeOS this is a very
 slow operation and could be speed up by caching the offscreen bitmap.

 There is a second menu item (ALT+S) for running the tests only in
 color_space B_RGB32.
