 Tracker lost files with non standard names while moving them from bfs -
 ntfs

 If filename uses <?> [ nonstandard naming for example "jakaś łódzka
 muzyczka.mp3" ] on bfs partition and i wanted to move it into ntfs
 partition tracker couldnt move it. Of course it monit that it cant move it
 [ because of nonstandard naming ] but after that tracker deletes the
 source file from BFS so file disappears from BFS and will not appear in
 NTFS partition.

 Definitelly a dangerous bug if you're keeping important files on bfs [
 documents ] and using non standard naming [ for example using polish
 alphabet ]
