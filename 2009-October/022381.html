 Opening Clip Up 2 (http://dev.osdrawer.net/projects/clipup) BeIDE project
 file with BeIDE on Haiku sends Haiku to KDL. Tested on VirtualBox 3.06
 running Haiku Alpha 1 on MacOS X and on real hardware. The project file
 works when running on BeOS R5 (under VirtualBox). Screenshot of the
 initial KDL and backtrace attached.

 Steps to reproduce:

 1)Mount BeOS R5 disk or unzip BeIDe from R5 devtools package

 2)Check out Clip Up source code from subversion

 3)double click trunk/ClipUP2_x86.proj file from the folder you downloaded
 the source code.

 4)When Tracker asks you to find application that can open the file, choose
 BeIDE

 5)Welcome to Kernel Debugging Land

 Expected behavior:
 BeIDE opens the project file (and if you don't have set up some symlinks,
 it complains of missing files and folders)
