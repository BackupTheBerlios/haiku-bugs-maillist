Comment(by mmlr):

 The file is basically a text file, but it contains 0 bytes at the end of
 each line. This technically makes it a non-text file, but most editors
 handle this case gracefully by stripping them or replacing them with
 placeholder characters. Just mentioning it, I don't know if it is a good
 thing to do this. The problem with replacing those characters with
 something else is that if you open such a file and then safe it again it
 will pretty much destroy the original data.
