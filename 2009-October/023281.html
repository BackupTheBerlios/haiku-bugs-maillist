Comment(by siarzhuk):

 I think it was not "broken" because worked for me already about 3-4 month.
 The things to check:

 1) The access rights for the key files - only owner should can read and
 write them;

 2) Host files must be generated with correct options:

 {{{
 ssh-keygen -t dsa -f /boot/common/etc/ssh_host_dsa_key
 ssh-keygen -t rsa -f /boot/common/etc/ssh_host_rsa_key
 }}}

 3) Password for the user must be set to non-empty - just use "passwrd"
 command to changing it from default empty one to something meaningful.

 Hope it helps. :-)
