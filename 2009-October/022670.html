 This is r33529.

 The default location for mails is ~/mail. Looks like by default these
 folders are created there: drafts, in, out. Some issues:

  * When saving a draft of an email, Mail creates a new folder "draft"
 (instead of the existing "drafts") to save the mail.
  * I have moved the folders of my mail accounts onto another partition. If
 there's no ~/mail folder, the draft can't be saved.

 Proposal: Save a draft in a "drafts" subfolder according to the used
 account, i.e. the location of the account as defined in the E-mail
 preferences. This would also mean a drafts folder for each account.
