 When I try to profile a media add-on with profile I get a General
 Exception Fault 0x0

 The stacktrace is like this:
 profiling_do_sample
 arch_debug_get_stack_trace
 get_next_frame_no_debugger
 arch_cpu_user_memcpy+01e  <-- Fault here

 Looking at the parameters passed the src ptr either contains 0 or a value
 > 0xfffffff0

 By placing a check in get_next_frame_no_debugger to return error for the
 values mentioned above I get profile data.

 The arch_cpu_user_memcpy adds a fault handler to trap these sorts of
 errors but it does not seem to be working for me.

 Tested on real hardware:
 Eee Box B202 Atom N270

 To reproduce try
 profile /system/apps/MediaPlayer <video file>
