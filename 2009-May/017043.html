Comment(by bonefish):

 pthread_getattr_np() is a non-standard GNU extension. I'm unsure whether
 we should provide it. Besides I wouldn't implement it by using the cached
 pthread creation attributes, but rather by collecting the info at that
 time. Most attributes we don't really support anyway -- so those can get
 the default values -- the stack address and size can be retrieved via
 get_thread_info().
