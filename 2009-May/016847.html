Comment(by umccullough):

 update-all is not a target, it's a build action. It's meant to be used
 with a build profile (very easy to setup in your UserBuildConfig).

 In order to use it, you would invoke with:

 jam -q @profilename update-all

 where @profilename represents a build profile you have declared to either
 build an image (either raw or vmware), build to a partition device, or
 install to BFS directory (the last option is only available from beos-
 compatible hosts)

 As stippi indicates, the update-all build action does have issues anyway.
