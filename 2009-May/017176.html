Comment:

 If you want jam variables to be overridable via environmental vars, you
 have to use the "?=" assignment operator in your UserBuildConfig -- it
 assigns only, if the variable isn't set yet. Build profiles always set the
 HAIKU_{IMAGE,INSTALL}_* variables, not allowing overrides. It should be
 possible to work around by saving the original environment variable and
 restoring its value after defining the build profile like this:
 {{{
 local savedInstallDir = $(HAIKU_INSTALL_DIR) ;
 ...
 DefineBuildProfile ...
 ...
 if $(savedInstallDir) {
   HAIKU_INSTALL_DIR = $(savedInstallDir) ;
 }
 }}}
