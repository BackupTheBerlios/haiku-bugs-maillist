Comment(by stippi):

 In r30557, I've now applied the enabling of the ATA stack via
 {{{HAIKU_ATA_STACK = 1 ;}}} in the UserBuildConfig also to generating the
 boot CD. You can create a boot CD with the command {{{jam -q haiku-cd}}}.
 *Only* this command will create a proper boot CD with write and attribute
 file system overlays, so that a ISO file-system, single-track Live-CD is
 created where programs think they can write to the CD. I don't know how
 you created the Senryu boot CD, but please check with a plain Haiku boot
 CD, in order to rule out any errors specific to your Senryu distro.
