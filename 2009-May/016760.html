 It is possible that systems are outthere that have an older gfx card, but
 a fairly recent monitor. In such a case the native resolution of this
 monitor can be higher than the highest available VESA mode in the card's
 BIOS.
 On top of that, it's very possible that the native resolution can be set
 by the haiku gfx driver. Since such a resolution will automatically also
 be preset in the 'vesa' config file, on next boot that non-existing mode
 will be set.
 The result is no signal (or hanging cards) during the boot and the boot-
 icons won't be visible.

 The kernel should check a vesa settings file against reported available
 VESA modes from the card at boot-time, and fall back to a lower mode if
 the requested mode is not there.
