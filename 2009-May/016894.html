Comment(by stippi):

 I looked at the BeBook documentation for DrawAfterChildren() again. It is
 unfortunately not clear, but I am pretty sure I checked the BeOS behavior.
 The documentation hints that you want to use this for the rare case where
 you want to paint *on* children, but it doesn't mention the fact that this
 is only possible if you also have the B_DRAW_ON_CHILDREN flag set.
 Anyways, although the documentation may be misleading, Haiku should be
 100% compatible with BeOS in this regard, I wrote a specific test
 application for this to cover various situations. DrawOnChildren() is
 never called if the flag is not set. It doesn't make any sense either,
 since if you don't paint *on* children, the drawing order is irrelevant.
