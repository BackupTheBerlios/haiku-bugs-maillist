Comment(by stippi):

 Would be a nice subject for some field research... how many apps out there
 treat Command-W to close the current *document* regardless of whether it's
 in a window by itself or inside a tab? From a user's point of view,
 closing the document instead of literally the window is more convenient,
 no? Since otherwise you have two shortcuts for the same thing in a single
 window application. As far as Terminal goes, it's like Firefox from a
 user's point of view. You can have multiple windows, but inside each could
 be more than one "document". For consistency with true single window
 applications, I'd vote to let Command-W always mean to close the current
 document, not the current window.
