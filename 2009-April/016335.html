Comment(by stpere):

 Okay, I'm glad to present this patch :) I'm proposing the patch for review
 and comments, there might be some rough edges and I'm willing to fix them.

 Highlights :

 * This patch (should) keep binary compatibility with apps

 * I introduced a new message (AS_GET_SYSTEM_FONT) in the app server
 protocol

 * There is a new constructor BFont(char *which) that fetch the current
 system font for this application..

 * The Decorator was reading its font from a DesktopSettings entity. It's
 no longer the case, it reads it from its associated application


 What is missing :

 * The feedback mecanism between currently running apps and the app_server
 to inform them that fonts are changing so that they can update their GUI
 (or ignore the change).  It could be a function hook or a message I
 suppose. Discussion on this topic is welcome.
