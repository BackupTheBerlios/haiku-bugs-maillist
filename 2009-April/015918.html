Comment(by stpere):

 I updated the patch to do/correct those extra things :

 * The revert function didn't revert the menu fonts.[[BR]]

 * The window wasn't centered at startup.  It seems to be a common issue
 with the layout engine. The Frame() function returns a rect 1x1 (the size
 specified in the BWindow constructor) in the constructor of the BWindow...
 and since we also call the _Center function at this time, we don't really
 center the window, the top left corner of the window will be at screen
 center.  I noticed that issue with other apps using the layout
 engine.[[BR]]
 Is there a way to force the window to compute its size before Show()?  I
 had to call Hide(); Show(); in the constructor to force the computation of
 the size. Yes, it's ugly..
