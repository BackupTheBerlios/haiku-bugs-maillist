Comment:

 Looking at BTab, it has no way of knowing which BTabView it belongs to, so
 it can't invalidate it. Therefore I assume that the application is
 supposed to do the invalidation, as I don't think the BTab can go just
 draw itself when the label is changed.

 I'll attach a patch that fixes the update issue with echo -e
 '\E]2;title\a'. Compared to the file posted earlier, it only invalidates
 the region that belongs to the tab whose title just got updated, which
 should reduce flicker and avoid re-drawing the whole tabview. It also
 doesn't lock the window as I don't think this is required with invalidate.

 With regards to the original suggestion, I think an in-place renaming by
 double-clicking the tab is a more sensible option than a menu option and a
 dialog. I'll see if I can implement this in the couple following days.
