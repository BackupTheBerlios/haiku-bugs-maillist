 Since a recent version of yasm is now in the buildtools/trunk, and there
 are several targets in the Haiku repo that require yasm to build for x86,
 I have put together a patch that builds yasm when the configure --build-
 cross-tools[-gcc4] is invoked.

 This patch has a couple issues, but does work on two of my Linux machines
 here, using both gcc2 and gcc4 cross tools.

 Issue 1: It prefixes the binary with i586-pc-haiku- when that is mostly
 unnecessary (since there isn't anything inherently haiku-specific about
 the yasm that is built).

 Issue 2: It will build yasm even for non-x86 targets when using --build-
 cross-tools-gcc4

 Issue 3: It hasn't been tested on other environments yet, nor on a native
 Haiku system... YMMV.
