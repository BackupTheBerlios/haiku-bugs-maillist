 I usually update my Haiku partition by copying the contents of a pre-build
 image to that partition under BeOS. (Like section 2.3 from http://www
 .haiku-os.org/documents/user/how_to_get_haiku_booted.) [[BR]]
 This time however, I reinitialized the partition under BeOS and then
 copied the new directory structure from the r29884 image to it.[[BR]]
 Now when I want to boot into Haiku, I get a message saying "Error loading
 OS; press any key to reboot."

 I suspect this is because BeOS's makebootable sets the partition boot code
 to look for the "/boot/beos/system/zbeos" file, while that one is now
 "/boot/system/haiku_loader". (BTW: Shouldn't that file be called
 "system_loader" instead?)

 I tried to run the makebootable from the pre-build image, but I get a
 message saying "sh: ./makebootable: cannot execute binary file".
