 One problem is BFS not supporting sparse files. The other is our file
 cache which isn't particularly clever with respect to large continuous
 reads/writes. So in this case it iteratively claims that many pages, which
 causes the page daemon to become very active trying to find pages that
 aren't desperately needed anymore.

 Anyway, the system is supposed to return to a usable state after a short
 while. Does it?
