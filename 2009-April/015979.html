 Considering that Terminal supports tabbed windows, should we add an option
 that can warn the user that by closing Terminal, they are about to
 forcefully abort running programs in other tabs?

 I'm hesitant to suggest this as default behavior, even though it parallels
 other programs.  As an example, StyledEdit ask for confirmation to save an
 unsaved work when closing.

 Maybe the default behavior can be to simply prevent Terminal from closing
 when a program is running.  Instead, Terminal can flash those tabs as an
 indicator of sorts.
