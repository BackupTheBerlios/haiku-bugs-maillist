Comment:

 That's because Tracker skips the pre-flight check when the op is a move
 and source / destination are the same volume. The rationale being, move on
 same volume is more or less an instantaneous op since it just requires
 removing/adding some dir entries and no actual data copy happens. As such,
 the pre-flight check that calculates the number of files to move would
 take substantially longer than the move itself. Will modify the dialog to
 omit the total count in those cases.
