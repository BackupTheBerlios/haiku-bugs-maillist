 Currently the haiku BFS is a little slow. My guess is that DMA transfers
 have not yet been enabled.

 There is a simple interim solution. By allowing the user to select a
 virtual memory path on a separate drive - preferrably on another bus
 entirely - disk swapping speed can be increased significantly. This is due
 to the fact that the physical heads in your drive need to change what
 they're doing to service a virtual memory call.

 By placing the virtual memory on a separate bus, optimum improvement is
 gained due to the fact that the PC's controller is now less loaded, and
 because there is now less data on the bus.

 I believe this would be a fairly simple change, and very worthwhile even
 in the long-term. Checking to ensure the drive is mounted and recreating
 virtual memory if this fails is about all the extra work required.

 No change would be noticed in systems with a single HDD, but then again no
 improvement is available from Haiku's ability to run SMP either if you
 only have one CPU ...
