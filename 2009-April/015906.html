 When checkfs runs, it will free up the memory of files that are already
 unlinked, but haven't actually been removed and freed yet (ie. there are
 still open file handles).

 Since it does not do this right now, checkfs might already do harm, as the
 blocks could then be reused immediately.

 We need some API in the VFS to allow for this.
