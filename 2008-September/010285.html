 > Yes, the fix removed the Intel Extended Partition, but also the NTFS
 partitions. See screenshot.
 >
 > Probably this happened because the NTFS-driver is missing, causing Haiku
 to think that the partitions don't contain a file system. But even if
 Haiku can't recognize the filesystem, the bootloader can boot that
 partition (by chainloading). So those partitions should also be listed in
 bootman.

 I hope this is fixed in r27503, if not someone should have a look at
 DiskDevice implementation why ContainsFileSystem() returns false for NTFS
 partition.
