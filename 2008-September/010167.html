 No, you misunderstood what was said. There is the normal "shutdown"
 command that functions the same as under a normally loaded Haiku and
 should be used to shut down Haiku in safemode. The exit command is a shell
 built-in to close the shell process, it's not a system tool. The shell
 that you see there is a normal bash, not a Haiku specific shell, so the
 built-ins work as they usually do in bash.

 when the shell exits. It states that the shell is restarted but doesn't.
