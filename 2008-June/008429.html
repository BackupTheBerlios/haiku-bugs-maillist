Comment(by mmlr):

 The errors that you get are simply because the working directory is not
 correct. If you simply drag an application to the Terminal it won't set
 the working directory. It will stay for example at "/boot/home". When your
 app now tries to read "graphics/xy.png" (a relative path, relative to the
 current working directory) it won't find it because it searches for
 "/boot/home/graphics/xy.png". When running apps from the Terminal like
 this you first cd to the directory where the app is located and then run
 it. When double clicking in Tracker, the working directory is
 automatically set. In general does this still happen?
