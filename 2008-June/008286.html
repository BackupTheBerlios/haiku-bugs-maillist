 Replying to [comment:6 umccullough]:
 >
 > Which is why many of us just chmod the necessary paths before jamming an
 image :)
 >
 > BTW, after you've been running your haiku builds as root - trying to run
 them as a normal user often fails to remove files created in the generated
 folder as root - so you'll want to chown those, or just remove them all
 and start clean.

 Take this:

 I got two mass storage controllers ...

 Haiku is on SCSI 0, Linux is on SCSI 3

 First time I boot Haiku gets /dev/sda, Linux gets /dev/sdb
 Next time I boot Linux might get /dev/sda and Haiku gets /dev/sdb

 So each time I test the "image" and boot back to update the sources I need
 to verify which one it got that time.

 In worst cases when it render the Linux partition unbootable, I have to
 boot from Live CD and I get Haiku on /dev/sda in one shell and /dev/sdb in
 another.

 God bless Ubuntu and ide-scsi ;)
