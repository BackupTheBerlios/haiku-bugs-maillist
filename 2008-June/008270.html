Comment(by bonefish):

 I'm a bit confused by this report. If the bfs_shell can't access the
 partition, how can anything be touched at all?

 Anyway, maybe there's a basic misunderstanding regarding
 HAIKU_DONT_CLEAR_IMAGE. It is not supposed to preserve any files on the
 image/partition. The only thing it does is disabling zeroing the
 image/partition. Zeroing usually makes no sense for partitions, but is
 useful for images when one intents to zip them. Regardless of this flag
 the file system is re-initialized on every "jam haiku-image".
