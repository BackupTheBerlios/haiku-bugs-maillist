Comment (by stim):

 I don't think this is a bug. It is just that bash uses tab for the
 complete operation. It doesn't matter whether the tab is pressed on the
 keyboard or pasted into the command line.

 I have confirmed that this behaviour occurs on linux as well, and I
 uncommented some tracing in the Terminal app and saw that Terminal was
 handling the tab character correctly. Another way to see that tab is being
 handled correctly is that command line completion is working in Terminal.
