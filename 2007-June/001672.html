 The driver is not included in the image by default.
 If you change the vmware configuration file (.vmx) by setting

 sound.virtualDev = "sb16"

 vmware should emulate a soundblaster16 device.
 BeOS R5 uses its sb16 when tested with vmware.
 On Haiku the device is published under /dev but there are no input/output
 nodes.
