 Replying to [comment:1 mmlr]:
 > That was most probably uncovered by r26492. The change itself is not
 problematic, but it enabled APIC timers on non-SMP systems. Probably there
 is a problem with the timer and/or APIC code in this case. Please try
 removing the "gAPICTimer" on line 41 of
 src/system/kernel/arch/x86/arch_timer.c to verify that. If this resolves
 the reboot problem we would need to look into better verifying the
 functionality of the APIC or review the APIC setup code.

 There seems to be some faulty BIOSes which cause problems when using the
 local APIC (as you already pointed out, btw). We should have a kernel
 setting and a safe mode option to disable the use of the local APIC.
