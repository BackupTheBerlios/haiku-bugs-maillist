 Terminal ignores <ctrl>D, so it's difficult to exit command line
 applications that expect EOF, for example Python interpreter.

 Apparently, because TermView::KeyDown() partially confuses this input with
 B_END, whose value is also 0x4.  But rawChar != B_END, so eventually it's
 just not handled by that case branch, and in the nothing comes out because
 it didn't go through the default case.

 The behavior can be fixed by adding an else {} clause, in the B_END
 branch, similar to the default branch, but that seems like kind of a band-
 aid.  <ctrl>D should probably not match B_END.
