 Attached is a patch that fixes the problem with the device type in the
 listdev output. See this line:
 {{{
 device Unclassified device (Non-VGA unclassified device) [0|0|0]
 }}}
 For some reason the relevant attributes are now B_UINT16_TYPE instead of
 B_UINT8_TYPE. It checked for the attribute name and type, so when the type
 did not match, the true value was not used, but the default 0 was used.
