Comment:

 The cause can be seen in the first screenshot at the top. Both the UHCI
 and EHCI controllers present have invalid IRQ assignments, which means
 that the BIOS didn't initialize interrupt routing for them. That means,
 this is in fact a duplicate of bug #5. If it works when you have a IDE
 disk connected this probably causes the BIOS to do some additional setup
 that it doesn't do when that's not the case. Maybe there are some USB
 legacy settings in the BIOS that you can try changing, otherwise there's
 not much you can currently do.
