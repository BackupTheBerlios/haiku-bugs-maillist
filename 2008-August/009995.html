 Since the rewrites in 26980/26981, I've been noticing some instability
 issues with the network stack, especially when larger amounts of data are
 involved. For instance, copying a few hundred megabytes of data via scp is
 guaranteed to at some point terminate the connection with the following
 message from sshd:

 {{{ Disconnecting: Corrupted MAC on input. }}}

 After something along those lines has happened at least once, connections
 seem to destabilize much more quickly afterwards, for instance I see
 Vision's network threads randomly go dead. I'm not sure what information
 would be most useful with respect to debugging at this point though.
