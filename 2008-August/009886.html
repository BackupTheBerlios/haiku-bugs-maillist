 The scsi scheduler threads suddenly sucks all cpu cycles (100% cpu usage)
 and in 30% of all cases causes a complete system freeze sooner or later,
 however never recovers from the 100% load.

 This happens on r27208 during installation from an usb stick (from which
 the system has been booted).

 It does take different amounts of time with each install trial, before the
 effect takes place.

 In 1 out of roughly 15 tries I actually managed a complete install before
 the system went bananas.

 This behaviour was not present with the previous builds (from before
 Sunday).
