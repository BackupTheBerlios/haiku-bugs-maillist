 In BeOS, all aplications inside /boot/beos/apps were symlinked inside
 /boot/apps. The reason was because /boot/apps was supposed to be a place
 where the user finds all applications available.

 Other than that, some R5 programs (like SpicyKeys) launch programs like
 the Terminal by path and tries to launch it as /boot/apps/Terminal, which
 obviously fails.

 I guess this is part of the R5 compatibility goal.
