 The question actually is, are two volumes with the same name allowed?

 Since the VFS layer contains a few function based on path value, like
 vnode_path_to_vnode and fd_and_path_to_vnode, having two volume with the
 same name can perhaps confuse those function.

 A little note though: in the rootfs partition, the volume is actually
 called Haiku1 and not Haiku like it is under the desktop directory.
 Beside, you can't rename a volume to Haiku as it result in
 "name already in use" error, which looks like an inconsistency.
