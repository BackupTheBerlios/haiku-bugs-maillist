Comment(by mmlr):

 That depends. The problem reported is that a certain program header was
 not understood (the negative value there translates to B_BAD_DATA and is
 returned in that case). So it is possible that there is a header that our
 runtime_loader doesn't support but objdump does. That's why I ask about
 the compiler version, as using our standard compilers this shouldn't
 happen.

 Checking elf specs the range 0x60000000 to 0x6fffffff where the indicated
 header (0x68003300) falls into are "reserved for OS specific semantics".
 So it seems that this is a header that was put there intentionally when
 generating the binary. Probably this is specific to another OS but was not
 disabled when compiling for Haiku. Objdump either does not understand and
 therefore skip it or it ignores it as OS specific.
