 When using TAB to switch fields, disabled fields are being taken into
 account and switched to. I don't remember how it was in R5, but
 considering the user can not do anything in a disabled field, they should
 probably be ignored.

 As an example, check the E-mail preferences app. When creating an account
 including an outbound chain, when configuring the outbound chain the
 fields username/password are disabled unless ESMTP is set but using TAB
 will switch to those fields.
