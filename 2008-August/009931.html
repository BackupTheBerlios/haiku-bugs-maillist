 Anyway, this has nothing to do with text encoding. We simply don't support
 all xterm escape sequences yet. E.g. we don't support the graphics
 character set, which is why there are "q", "x", "l", "j" instead of lines.
 So the open "Windows" menu looks at good as it gets ATM. I don't see what
 the box to the left is supposed to be, but I guess the remainders of some
 menu. So maybe some positioning escape sequences are used, that we don't
 support yet. If you can find out which ones, it's probably not so hard to
 add them.
