#1247: Tracker multiplies calls (open,stat) per open window
------------------------------------+---------------------------------------
 Reporter:  mmu_man                 |       Owner:  axeld         
     Type:  bug                     |      Status:  new           
 Priority:  low                     |   Milestone:  R1            
Component:  - Applications/Tracker  |     Version:  R1 development
 Platform:  All                     |  
------------------------------------+---------------------------------------
 When volume usage bars are enabled, Tracker regularily (several times /s)
 forces recreating BVolume objects, implying multiple calls to open() and
 stat() on each volume's root folder.
 While it's needed only for the desktop window, it is actually done by each
 open window, so if you have 10 open windows you get 30 stat() per second,
 which on uncached network fs is quite wasteful. Also polutes syslog.

 It was so at least in Zeta's Tracker last time I checked, didn't try with
 svn but I don't think it's ever been fixed.
