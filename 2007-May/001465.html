 This currently appears to be a seperate issue, but may be related to
 ticket #1240.
 This was originally filed under ticket #1231 as an issue with the PCL6
 driver, but after review appears to be related to Haiku's implementation
 of BTextView and/or BScrollView.

 How to reproduce:
 When adding a PCL6 printer, an additional dialog shows up to select the
 protocol. If user scrolls the description text down for PCL 6 Protocol
 Class 1.1 and then resizes the dialog, the description text does not
 adjust vertically. The result is the top text is cut off and can not be
 reached without making the dialog smaller again.
