#1234: adapt our build system to distro guidelines
--------------------------+-------------------------------------------------
 Reporter:  wkornewald    |       Owner:  bonefish      
     Type:  enhancement   |      Status:  new           
 Priority:  critical      |   Milestone:  R1            
Component:  Build System  |     Version:  R1 development
 Platform:  All           |  
--------------------------+-------------------------------------------------
 Since others may not use our logo and trademarks we need to extend the
 build system. The preferred solution is probably to have good defaults
 (i.e.: neutral names and logos) and require us and "Haiku Compatible" 3rd-
 party distro makers to set a few build options. By default, no Haiku logo
 and trademarks would be enabled (remove from AboutHaiku and boot screen,
 etc.). AboutHaiku is renamed to AboutSystem and the Deskbar menu item is
 labeled "About This System...".

 The build system would have the following options:
  * default (no logos/trademarks)
  * HaikuCompatible (enables a "Haiku Compatible" logo in boot screen and
 AboutSystem)
  * Haiku (official build; enable Haiku logos and trademarks in boot logos
 and AboutHaiku)

 I'm not sure if we should rename "About This System..." to "About
 Haiku..." in the official "Haiku" build.
