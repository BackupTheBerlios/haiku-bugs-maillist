#1543: Check libroot for Correct Use of errno (easy)
-------------------------------+--------------------------------------------
 Reporter:  bonefish           |       Owner:  axeld         
     Type:  bug                |      Status:  new           
 Priority:  normal             |   Milestone:  R1/alpha      
Component:  System/libroot.so  |     Version:  R1 development
 Platform:  All                |  
-------------------------------+--------------------------------------------
 While BeOS API functions usually return error codes directly, POSIX
 functions just indicate an error by a special return value (often -1 or a
 NULL pointer) and set errno to the actual error code. Our libroot code
 (src/system/libroot, particularly the posix subdirectory) needs to be
 reviewed in this respect.
