Comment:

 I don't know if this "bug" describes USB or PS/2 keyboards.

 We have to keep in mind that some people might need long delays and low
 repeat rates in order to use the keybaord at all.

 The PS/2 driver is using the PS/2 keyboard reset standard settings, that
 also BeOS R5 used. That is delay = 0x01 (500 ms), rate = 0x0b (10.9
 chr/sec). Of cause you can overrule this with the Keyboard preferences.

 While I use the fastest settings myself, I think we should keep the
 defaults slow.
