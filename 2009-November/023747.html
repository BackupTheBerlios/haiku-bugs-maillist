 gcc2, rev 34013, Virtualbox 3.0.10.

 Our .vmdk file no longer works in Virtualbox. It immediately fails with
 the following error, without showing any boot splash:
 {{{
 FATAL: No bootable medium found! System halted.
 }}}

 However, converting our raw .image file to a .vdi file makes Haiku boot
 fine. The command used for converting the image was:
 {{{
 VBoxManage convertfromraw -format VDI haiku.image haiku.vdi
 }}}
 [[BR]]
 I am not sure whether this is due to changes in Virtualbox, a regression
 in Haiku, some old Haiku bug triggered by the new version of Virtualbox,
 or something else entirely. Suggestions for further testing welcome.
