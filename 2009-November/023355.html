Comment(by bonefish):

 I'd be in favor of introducing a {{{thread_id
 get_pthread_thread_id(pthread thread)}}} at least, which would make it
 possible to use the Haiku thread API in the first place. A function for
 the reverse conversion would make sense as well. Though I'd suggest a name
 that implies that the thread then has to be handled with the pthread API
 (e.g. {{{status_t convert_to_pthread(thread_id thread)}}}), since
 otherwise the user would be leaking the pthread structures.

 Regarding the {{{pthread_attr_{get,set}startsuspended()}}} suggestion,
 does something like this exists in other OSs?
