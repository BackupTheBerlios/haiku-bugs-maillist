 This is r33939.

 The context menus on the headings of attribute columns in Tracker list
 views should behave exactly like any other menu:
  * It should stay open when it is single clicked until an entry is
 selected, which closes it automatically.
  * If you click and hold, you can navigate to an entry, release the mouse
 button and the menu closes automatically.

 At the moment, only the latter works.


 Of course, having a method to select more than one entry without having
 the menu close automatically, would be desirable for the whole menu
 system. Think of starting several apps from Deskbar or going through
 several fonts in an wordprocessor. That's a separate topic, however.
