#1817: Tracker Trash preferences / Ask before delete does not work
----------------------------+-----------------------------------------------
 Reporter:  hma             |       Owner:  stippi        
     Type:  bug             |      Status:  new           
 Priority:  normal          |   Milestone:  R1            
Component:  User Interface  |     Version:  R1 development
 Platform:  All             |  
----------------------------+-----------------------------------------------
 Tracker does not show conformation dialog for moving files to trash or
 emptying trash.

 In Tracker Preferences / Trash there is a checked check box "Ask before
 delete", but no asking is done.

 I would also suggest following usability improvements:

 "Don't move files to trash" checkbox does not inform the user what is the
 alternative. Using radio buttons and spelling out the alternative would be
 better. Like this:

 ---When files are moved to trash-------------
 |
 | (*) Keep files in Trashcan.
 | ( ) Delete files permanently.
 -------------------------------

 This way the setting is more conceptually compatible with Tracker's "Move
 to trash" command, and unwise setting sounds bad. :)

 It is unclear if "Ask before delete" means getting confirmation dialog
 before moving item into trash or  when trashcan is emptied. I would
 suggest having a separate checkbox for both, so that experts are free to
 shoot themselves in the foot and common people do not screw up when
 accidentally pressing "Delete" key. Like so:

 --Ask before... ------------------------
 |
 | [x] moving files into trash.
 | [x] deleting files permanently.
 -----------------------------------------
