 Binutils 2.17 objdump (070218 gcc package from BeBits) crashes when
 disassembling (gcc 2.95.3 generated) objects with debug info (objdump -D
 --demangle ...). Can be reproduced under both Haiku and BeOS.

 I tracked down one problem to a dereferenced NULL pointer in
 opcodes/i386-dis.c:print_insn() (first putop() call). Apparently the bug
 has already been fixed in CVS version 1.132 (2007/08/02 00:40:02) of the
 file.

 I get a different stack trace for another test file (libperl.so compiled
 with "-g") though, so it seems not to be the only problem.

 Update to newer binutils recommended.
