#1773: $0 is set incorrectly
----------------------+-----------------------------------------------------
 Reporter:  andreasf  |       Owner:  axeld         
     Type:  bug       |      Status:  new           
 Priority:  normal    |   Milestone:  R1            
Component:  System    |     Version:  R1 development
 Platform:  All       |  
----------------------+-----------------------------------------------------
 $0 should contain the (relative) path to the executable, not the command
 used to call it.

 The following test script:
 {{{
 #!/bin/sh
 echo $0
 }}}
 called with `test.sh` prints {{{./test.sh}}} on BeOS but {{{test.sh}}} on
 Haiku.

 This seems to keep aclocal et al. from working, since $0 returns
 {{{aclocal}}} but the path to call itself is e.g.
 {{{config/bin/aclocal}}}.
