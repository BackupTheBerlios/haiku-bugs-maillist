 As a testbed for kernel development, profiling, etc. a virtual kernel
 running on top of the actual kernel would be beneficial. Also having it
 run on top of ie. Linux or BeOS would be nice, if possible, especially as
 long Haiku is not self hosting.

 Jonas Sundström was kind enough to collect some more informations:

 "Reduces the engineering cycle time for testing new kernel code
  from 5 minutes to 5 seconds."  -- Matthew Dillon
