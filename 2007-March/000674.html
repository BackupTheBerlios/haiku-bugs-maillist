Comment (by axeld):

 You're misusing gprof :-)
 You're supposed to run the executable, and quit it (killing it won't
 help), then it writes out a file called something like profile*** - and
 *that* file is the one you have to use gprof on.
 Because of these semantics, you can more or less only use profiling in the
 app_server test environment. Or write your own profiling code (instead of
 using the one provided by i386-mcount.o).
