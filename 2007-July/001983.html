 If the /boot/home/people folder doesn't exist, the People file's default
 name is corrupted.

 When the dialog is closed, and Save is invoked again, it works fine (the
 folder was created during the first try). It also works if the People
 application or the whole OS is restarted.

 See attached picture for reference.

 ----
 To reproduce:
  - Make sure the /boot/home/people directory doesn't exist, and People
 isn't running.
  - Start People.
  - Enter a name.
  - Select File -> Save.
