Comment (by stippi):

 I would assume it locks up the input_server. To check this, run something
 that displays an animation, like the bitmap drawing demo, or just see if
 the clock keeps updating. Several possibilities:

 * The input_server restarting is not implemented correctly
   - input_server does not restart correctly - or
   - app_server does not pick up the new input_server, maybe even locks up

 * Having UniversalScroller in the input_server add-ons folder crashes the
 input_server, even on startup. To check this, place the add-on in the
 folder from another OS and boot Haiku with the add-on already in place.

 Can you investigate in these directions and update the bug report? That
 would save some time. Thanks a lot!
