 This relates to this thread I started relating to kernel panics in R32959
 http://www.freelists.org/post/haiku/bug-tracker-advice-plse
 I was getting consistent panics reporting:
 "page fault but interrupts were disabled touching address 0x00000010 from
 eip 0x008428cc
 Thread 25 "media checker" running on CPU0"
 whenever the cd drawer closed. Only on the master drive.

 On advice in that thread that much had changed in the disk controller I
 upgraded to R1/alpha1 Revision 33109 before trying again.

 Now, I get no kernel panics but the master drive is not seen at all -
 disks do not appear in the Mount menu, nor does /dev/disk/atapi/1/master
 appear in Drive Setup. The slave drive is seen and works properly.

 Also, I assume this is related: At the point where I had written an
 install disk of the R1/alpha1 ISO I tried to install it from the master
 drive; the machine booted from the disk but then immediately took me into
 the existing R32959 system with no further user input. I had to put the
 disk into the slave drive for the install process to work as it should.
