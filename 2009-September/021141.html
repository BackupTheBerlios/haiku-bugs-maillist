 Can you please try the checkfs command on the partition in question? I
 suppose the file is deleted, but not freed because it is still in use.
 Then on reboot this gets lost (as our journal format doesn't allow to
 track this situation). Running checkfs should free up these blocks and
 show a corresponding block number being freed.
