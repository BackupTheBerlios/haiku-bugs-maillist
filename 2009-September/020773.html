 This is most easily observable by running Tracker in the debugger. When
 Tracker creates a window, it is initially named TrackerWindow and then
 updated to indicate the folder name once the PoseView is added to it,
 which results in BWindow also issuing a rename_thread() to match the
 thread name to the new window title. However, the Debugger never notices
 this name change and simply shows it as w>TrackerWindow the entire time.
