#4631: Two Haiku partitions cannot change settings independently
-----------------------+----------------------------------------------------
 Reporter:  HAL        |       Owner:  axeld    
     Type:  bug        |      Status:  new      
 Priority:  normal     |   Milestone:  R1       
Component:  - General  |     Version:  R1/alpha1
Blockedby:             |    Platform:  x86      
 Blocking:             |  
-----------------------+----------------------------------------------------
 If I boot up Haiku r33234 and the other HD with Haiku R1/alpha1 enabled
 from the BIOS, it takes on all the settings of Haiku R1/alpha1. The About
 This System says R1/alpha1 r33109. The partition size is slightly larger
 for Haiku on the other partition and when I right click the Haiku drive
 desktop icon then click get info it shows the drive size as if it was the
 other partition.
 If I boot with the the other HD that has Haiku, disabled from the BIOS, in
 About this system it shows r33234. I can then change the settings
 independently.
 If I booted up with Haiku r1 alpha1 on the other enabled and then tried to
 change settings in r33234 then rebooted into Haiku r1 alpha1, I would find
 the changes reflected there.
 I tried to get a screenshot of DriveSetup in R1/alpha1, but could not save
 it anywhere because the partition with r33234 was enabled. Clicking the
 save button would not do anything.
 To see this bug:
 1. Have two SATA (or maybe PATA) drives and one PATA drive
 2. Have Haiku r33243 on first partition of second SATA drive.
 3. Have Haiku R1/alpha1 on first partition of PATA drive
 4. Boot up r33234 PATA drive/R1alpha1 enabled.
 Result is:
 Haiku has exactly the same setting as Haiku R1/alpha1 r33109. Have a look
 in DriveSetup and there is no partitions shown for the second SATA drive.
 Partitions are shown for the PATA drive as if it is the one booted but it
 is not.
 Expected:
 after booting Haiku on Second SATA drive to have different colors settings
 and its own partition info in DriveSetup.
 Also all the other partions are missing on second SATA drive:: linux
 reiser partitions and a swap partition.
 If I do the same steps above with the PATA drive disabled from BIOS,
 everything behaves as expected. The proper Haiku partition is visible in
 DriveSetup along with the Linux partitions.
