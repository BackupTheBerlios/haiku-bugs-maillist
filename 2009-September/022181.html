Comment(by dlmcpaul):

 With the auich driver there are some settings you can play with.

 # Settings file for the auich driver
 #
 # This file should be moved to the directory
 # ~/config/settings/kernel/drivers/
 #

 #sample_rate    48000
 #buffer_frames  256
 #buffer_count   4
 #use_thread false

 I would put this line in the place mentioned above and uncomment
 buffer_frames and buffer_count.

 Then make changes like increase buffer_frames to 4096 say and reboot.  See
 if it makes any difference.

 I think buffer_frames is how many frames can be played from one buffer and
 buffer_count is how many buffers there are available.

 cracks and pops I think would be improved by increasing buffer_frames.
