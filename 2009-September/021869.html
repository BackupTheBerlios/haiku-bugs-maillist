Comment(by leszek):

 Ok I found out something new.
 The file I uploaded as an example was created with ffmpeg but gave out the
 error message : wrong buffer size .
 Its a known bug in ffmpeg together with libmp3lame 3.98
 from '''howto-pages.org/ffmpeg''':
 {{{
 The error message about the output buffer being too small is nothing
 important. It's merely a result of the way data is passed between ffmpeg
 and the external codec, LAME.
 }}}
 So the file plays fine on various linux mediaplayers except vlc which
 regardless of its platform plays only the video and gives out an error
 about a corrupt avi header.
 After searching through the web and of course my hd I found various of
 those files.
 Sadly I think there is no easy fix for this.
