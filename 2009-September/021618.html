 This is an update from the report yesterday on the dev mailing list.
 Having waited a bit longer than previously this would appear not to be a
 complete hang but still unexpected behaviour.

 Phoenix BIOS
 BIOS Version R0040J1
 EC BIOS Version RK040J1
 Video BIOS Version 5.44.02.23.22

 Steps to reproduce

 If I exit Haiku via :-

 1) Leaf Menu
 2) Shutdown
 3) Restart System

 Observed behaviour:
 4) Haiku closes down.
 5) BIOS logo comes up
 6) System pauses for almost a minute at this point. Esc and F2 take it to
 the textmode bios splash but will not actually proceed to the bios setup
 menus until the minute is up.

 Expected behaviour:
 After step 3 :-
 4) Haiku should close down.
 5) System restart and show the BIOS logo
 6) A short pause about 2-5 seconds
 7) Proceed to bootman.

 During step 6) I would expect the BIOS screen to respond to F2 keypresses
 by taking me to the BIOS setup menus almost immeadiately.
