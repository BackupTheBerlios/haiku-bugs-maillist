 Blocking:                 |

 When creating a new directory (e.g.: net) in /boot/home/config/add-
 ons/kernel/drivers/dev/ following message appears in the syslog:

 ''KERN: Could not load kernel add-on "/boot/home/config/add-
 ons/kernel/drivers/dev/net": Is a directory''

 This is a reproducable bug, at least in R1/alpha1 (r33109).

 The message in the syslog seems to be the only side effect of this bug.

 I would expect the kernel not trying to load a directory as a driver at
 all. Thus no message that made me think, something went wrong with the
 driver installation.
