 For the Control-Space key sequence, the bytes array in the BMessage
 contains an ASCII NUL.  Attached is a patch to the input_server keyboard
 driver, the BMessage FindString method, and the BWindow DispatchMessage
 method which insures that the KeyDown and KeyUp methods for Control-Space
 will be invoked with the correct length.

 The inline loops here are a little cumbersome, perhaps BString should have
 a method for setting the contents of the string to binary data which may
 contain NULs?
