Comment(by zooey):

 You can reproduce the problem with VMware-player on Linux, too, by adding
 these two lines to the *.vmx file:

 {{{
 RemoteDisplay.vnc.enabled = true
 RemoteDisplay.vnc.port = 5900
 }}}

 After starting the VM and using vncviewer to connect to localhost:5900,
 the VNC viewer window comes up with the Haiku desktop. Mouse behaviour in
 that window is indeed broken (while it works fine in the VMware-player
 window).

 No idea if this helps with tracking down the problem: haiku mouse
 behaviour is ok when using the same image over VNC in qemu-kvm (with the
 same VNC client). In that setup, the mouse is too fast (i.e. the VNC
 cursor and haiku's cursor move at different speeds), but that can be
 remedied by deactivating mouse acceleration in haiku's mouse prefs.
