Comment(by mmlr):

 I've adapted the specs in r29297. Note that just adding -shared to the
 linkspecs isn't enough, but also excluding start_dyn.o is required.

 About the shared libstdc++. It will be provided as a binary when the
 package is updated. I don't think it's really worth the effort to build it
 as shared when building the cross-compiler. Therefore it's not really
 necessary to hack in the Haiku specific files into the gcc build system.

 GCC as well as Haiku can be built just fine under Haiku, so it's not
 really a problem to just build the shared libstdc++ afterwards. The GCC4
 package will provide it once it is updated.
