 When trying to compile latest scummvm and trying to run it I came across a
 problem with the missing header GNU_STACK.
 Looks like this is generated by nasm when building the scaler graphics for
 scummvm. The lines where they are created are:

 %ifidn __OUTPUT_FORMAT__,elf[[BR]]
 section .note.GNU-stack noalloc noexec nowrite progbits[[BR]]
 %endif[[BR]]


 If I put a # to prevent them from being included scummvm still compiles
 fine and doesn't have the extra header anymore (found this after a tip
 from DeadYak with objdump).

 Scummvm runs fine without it so question is, is it a scummvm or a haiku
 problem?
