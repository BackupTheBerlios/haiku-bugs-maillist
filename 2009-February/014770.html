 When new folders are created for an IMAP account to match those existing
 on the server, they currently show up by default like a normal tracker
 folder, which is to say filename / last modified by / size and such, which
 aren't generally as useful when trying to use MDR as intended. It would be
 nice if mail_daemon could preconfigure the directories such that a more
 suitable set of attributes appear when first opened.
