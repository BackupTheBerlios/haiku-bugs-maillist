 I'm not sure how to provide any further details on this bug as I can't do
 a stack trace or anything.

 The motherboard is a ASRock Conroe1333-DVI/H that has a GMA950 chipset
 with a riser card in the PCI-E slot that has a DVI output.

 lspci from Linux gives the controller as:
 00:02.0 VGA compatible controller: Intel Corporation 82945G/GZ Integrated
 Graphics Controller (rev 02)

 When I boot Haiku, after the very last boot icon the monitor switches off
 - no signal.  Caps/numlock don't seem to work on the keyboard when the
 monitor is off, so I assume it's frozen as well.

 Booting in safe graphics mode works fine.

 I've tried removing the DVI riser card and just using VGA, but I get a
 blank screen instead of the boot icons.

 Is there a log file or something that I can check after rebooting in safe
 graphics mode?
