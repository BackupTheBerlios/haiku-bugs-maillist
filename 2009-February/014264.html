#3412: stdio library code dishonors BUFSIZ for setbuf()

 If a program calls:

 {{{
 char buf[BUFSIZ];

 setbuf(stdout, buf);

 then call many:
 putc(char, fp);
 }}}

 stdio still tries to write more than BUFSIZ (1024) bytes into the buffer
 and overwrites it. This causes other global varibables in the
 binary (located past buf) to be overwritten.

 This bug seems to be related to bug #3411
