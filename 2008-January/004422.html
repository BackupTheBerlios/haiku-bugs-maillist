Comment (by fyysik):

 Replying to [comment:1 korli]:
 > Obviously a keymap switcher should integrate well with switching input
 methods.

 This is bit unclear sentence. IMHO, from user POV, deskbar replicant for
 switching keymaps for "phonetical" alphabets, shoudln't contain choice for
 input methods.
 Maybe only in additional advanced section which allows to set different
 shorcuts to activate this or that InputMethod. But in this case deskbar
 idndicator  would contain second display field, separate from base keymap
 field.
 Keymap is primary thing in relation to Input Methods. To be more clean -
 user of input methods need ability to set basic "phonetical" keymap
 (variation of latin in simplest case) to that which suits his/her
 "physical" keyboard better. And Input Method  - to use just that
 keymapping.
 Only thing I can propose for better integration is keymap locking (by
 blocking switcher) while input method is activated.
