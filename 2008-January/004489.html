---------------------------+------------------------------------------------
 Reporter:  stippi         |       Owner:  axeld         
     Type:  bug            |      Status:  new           
 Priority:  normal         |   Milestone:  R1/alpha      
Component:  System/Kernel  |     Version:  R1 development
 Platform:  All            |  
---------------------------+------------------------------------------------
 * Mount a BFS partition in Tracker
 (Right clicking this volume's icon has the usual popup menu.)
 * Unmount the partition
 (The volume's icon will stay on the desktop, right-clicking it will show
 the folder popup menu.)
 * Try to invoke Tracker's mount menu -> Tracker will now hang.
 * Killing Tracker kills the complete team, but the windows will stay on
 screen. Looking at the kernel team shows a thread named Tracker.

 More info: When mounting/unmounting a partition using DriveSetup,
 DriveSetup will hang at unmounting the same as Tracker. Killing it removes
 the team completely, but the windows stay on screen and there is a thread
 called DriveSetup in the kernel team. Trying to invoke the mount menu from
 Tracker results in Tracker hanging too.

 Can't investigate further with kernel debugger atm, because of USB
 keyboard, but this is 100% reproducible for me.
