Comment (by bonefish):

 Should be fixed in r23544. Please test.

 Regarding copying test files to the image, you probably already know that
 you can copy individual files using the AddFilesToHaikuImage rule. Copying
 directories recursively is not provided by a rule. You can define the
 build system variable HAIKU_IMAGE_LATE_USER_SCRIPTS to refer to a shell
 script that will be sourced by the build_haiku_image script. In this
 script you can customize your image/installation at your heart's desire.
 E.g. copying a directory to the image would look like this:
 {{{
 $cp -r "${sPrefix}/path/to/directory" "${tPrefix}home/whereever"
 }}}
