Comment (by korli):

 This patch fixes the bad behavior on 16 bits colorspaces. Rendering is
 really done with a 16 bits colorspace buffer which is logical.
 Though on BeOS, it seems the rendering is done with a 32 bits colorspace
 buffer, which would explain why GLTeapot behaves correctly there.
 I think this problem is a bit implementation dependent, and GLTeapot bases
 its behavior on the implementation: this would be a good reason to patch
 GLTeapot.
