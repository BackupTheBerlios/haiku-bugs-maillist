Comment (by axeld):

 Whatever get_system_info() returns is a momentary report on resource
 usage. The only thing safe about this is that if you'd crash in that
 function (KDL), the results would accurately describe the current status
 quo. You also cannot measure how much memory a specific application is
 actually using vs. how much it has reserved (ie. cached).

 Since memory_monitor.cpp does not make any assumptions about the values it
 prints out, it's of course okay. The number of free pages only count those
 pages that are actually not used by the system at all (ie. are either in
 the free or clear page queue). HTH :-)
