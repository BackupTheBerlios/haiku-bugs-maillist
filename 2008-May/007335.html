Comment (by bonefish):

 I'm actually not quite sure, whether I find it a good idea to use
 find_directory() in system components like the kernel or the runtime
 loader. They just complicate things without real benefit. I mean, the
 whole point of using find_directory() in applications is to still work as
 expected when the system's directory layout changes. But these components
 *are* the system. I think I'd rather use a common single header file that
 defines the paths.
