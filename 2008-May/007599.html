Comment (by mmlr):

 I'd tend to classify this as invalid. The cp command has never copied BFS
 attributes and IMO should not either (it's a handy way of getting rid of
 attributes). So this is expected behaviour. There is "copyattr -d" that
 copies file data and attributes and should be used to achieve attribute
 and data copies from the command line. But I won't decide on that, waiting
 for other opinions.
