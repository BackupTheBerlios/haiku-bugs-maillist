Comment (by stippi):

 I did fix the flickering for VESA and I have a plan for how it could be
 fixed it for accelerated drivers: By using a back buffer that is used only
 for the region touched by such transparent overlays, separating drawing
 commands that arrive intersecting with this region into a part that is
 drawn in software (to the back buffer) and another part which is rendered
 to the front buffer (accelerated for certain commands). The part that
 touched the back buffer is then composited with the overlay and copied to
 the front buffer. This would eliminate flickering in a way similar to the
 current VESA mode while still using acceleration. If someone wrote a
 driver for AMD X1400 hardware, that would certainly boost my motivation to
 work on this... :-)
