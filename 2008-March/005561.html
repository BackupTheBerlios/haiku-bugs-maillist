Comment:

 Axel,

 I have been facing this issue too in a triple-boot configuration with
 Ubuntu, BeOS and Haiku. In BeOS the time looked good but is off in Haiku,
 leading to issues when accessing a common BFS volume, i.e. for `make` (as
 mentioned recently on the list).

 The natural way to fix this would be to offer a choice of storing as local
 or GMT, like Linux did until recently. It should default to GMT and offer
 local time for Windows compatibility only. But currently it seems that
 only displaying GMT time is possible.

 A workaround might be to add NTP support to Haiku so that at least the
 time is adjusted to displaying and using the correct time even if storing
 it in different ways?
