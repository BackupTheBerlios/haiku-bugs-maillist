 current output directory (.../gcc-obj/gcc/). According to the kernel

 "bfs_read()..." line in the serial debug output. I suspended and uploaded
 the virtual machine state (512 MB RAM, 1 GB disk):

   http://tfs.cs.tu-berlin.de/~bonefish/private/haiku/CrashOMatic-non-
 existent-cp.zip
